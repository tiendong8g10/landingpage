# Design Document: Landing Page Hoi trai tong quan 2026

Date: 2026-02-27
Status: Approved by user

## 1. Objective
Build a single-page landing website for the 2026 military camp communication program, with a newspaper-like reading flow and light modern motion. The site must present 4 primary content sections and image overlays.

## 2. Scope
In scope:
- Single-page vertical landing layout
- 4 main sections in fixed order
- Intro hero with official title/subtitle and provided lead text
- Image galleries sourced from the 4 DOCX files
- Click-to-open image popup/overlay with caption and prev/next controls
- Responsive layout for mobile/tablet/desktop
- Deploy-ready for GitHub -> Vercel workflow

Out of scope:
- SEO/meta optimization (explicitly removed per user request)
- CMS/admin editing
- Backend or database

## 3. Information Architecture
1. Hero
- Main title: "Hoi trai tong quan nam 2026 - Chu de \"Tuoi tre Xuan Thoi Son - Tiep buoc cha anh, len duong giu nuoc\""
- Subtitle: "Tieu trai Pham Van Sang"
- Intro paragraphs: 2 provided bullets converted into readable paragraphs

2. Section A: Quan chung Phong khong - Khong quan
3. Section B: Cac hien vat
4. Section C: Khong gian van hoa Ho Chi Minh
5. Section D: Cac hinh anh tuyen quan

Each section includes:
- Section heading
- Optional short section intro
- Image grid with captions

## 4. Visual Direction
- Tone: ceremonial/military + subtle contemporary polish
- Reading style: longform news article rhythm
- Color tokens:
  - Background paper: #F7F3EA
  - Primary text: #1F2937
  - Accent red: #9B1C1C
  - Accent gold: #C9A227
  - Deep military green: #1F3A2E
- Typography:
  - Heading: Merriweather (serif)
  - Body: Be Vietnam Pro (sans)
- Motion:
  - Light reveal-on-scroll
  - Subtle image hover feedback (no aggressive scaling)
  - Overlay transitions around 180-220ms
  - Respect prefers-reduced-motion

## 5. Interaction Design
- Sticky top nav with anchor links to 4 sections
- Gallery cards are clickable and keyboard-focusable
- Overlay behavior:
  - Open on card click
  - Prev/Next navigation
  - Close on ESC, backdrop click, and close button
  - Body scroll lock while open

## 6. Content/Data Strategy
- Source files:
  - Quan chung Phong khong.docx
  - hien vat.docx
  - KGVH HCM.docx
  - tuyen quan.docx
- Extract images from each DOCX and map into matching section order.
- Captions are assigned by sequence in each DOCX; where caption text is missing, fallback caption is generated by section and index.

## 7. Technical Plan (High Level)
- Stack: Vite + Vanilla JavaScript + CSS
- Files:
  - index.html
  - src/main.js
  - src/style.css
  - src/data/content.js
  - scripts/extract-docx-assets.py
  - public/images/... (generated)
- Render UI from structured JS data object.

## 8. Reliability and Accessibility
- Alt text for all meaningful images
- Visible focus states for keyboard users
- Color contrast maintained for body text
- Image load fallback UI
- No horizontal overflow on mobile

## 9. Verification
- npm run dev starts successfully
- npm run build succeeds
- Manual checks:
  - anchor navigation works
  - overlay open/close + keyboard controls work
  - responsive layout at 375/768/1024+ widths

## 10. Deployment Workflow
- Local code -> GitHub push -> Vercel project import/deploy
- Vite default static output used by Vercel
